Why Astro: it's the only stack that natively handles per-type layouts (Plan vs ADR vs Review) via content collections, supports React islands for search/filters, and lets the Crystal palette ship as plain CSS without fighting a framework. mkdocs is faster to set up but you'd hit the wall when designing the Plan card with task progress. Vite is what Astro is built on top of — no reason to redo that work.
Default workflow if A: open one terminal with npm run docs:dev while editing; Claude edits MD, browser auto-refreshes. For "static viewing" or sharing, run npm run docs:build any time — output is plain HTML you can open via file:// or commit alongside the repo.
